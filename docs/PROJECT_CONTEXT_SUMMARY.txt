二、已完成的主要功能

1. 标注逻辑改造

- 从“整块 tag”改成“自然语言 caption + 可拆分短语”模式。
- 每个短语都能单独编辑、删除、替换。
- 相关逻辑主要在 core/dataset_workspace.py。

2. Viewer 与 Caption 编辑

- Viewer 区支持直接编辑 caption。
- 支持 chip 化短语编辑。
- 支持追加短语、批量增删改、翻译、排除当前条目。

3. 快捷标注

- 单击直接追加到当前 caption。
- 双击直接编辑。
- 拖拽排序。
- 单独存储长期保留。
- 默认收起。
- 输入内容所见即所得，不自动补标点。

4. 工作区能力

- 支持加载 control1 / control2 / control3 / result。
- 支持输入父目录后在网页里浏览并选择子目录。
- 支持将额外数据集合并到当前工作区，不清空已有内容。
- 支持重扫工作区。

5. 顶部主操作区

- 已整理成：
  - 选择标注引擎
  - 标注当前
  - 批量标注
  - 标注配置
  - 停止
- 已实现“选后端后，外部按钮直接调用”的模式。

6. 标注后端

- 支持本地 Qwen。
- 支持 Ollama。
- 支持 OpenAI 兼容 API。

7. OpenAI 兼容 API 配置

- 支持 API URL。
- 支持 API Key。
- 支持手动刷新模型列表。
- 支持自定义模型搜索/选择菜单。
- 选择模型后菜单会自动收起。
- API Key 已做本地持久化，存储在浏览器 localStorage。

8. 项目模块

- 支持保存当前项目。
- 支持打开历史项目。
- 支持项目缩略图。
- 支持重命名 / 删除 / 搜索 / 排序。
- 项目数据落在 datasets/projects。

9. 图像处理

- 支持批量生成预处理工作集。
- 使用 Lanczos 缩放 + 中心裁切。
- 默认 4 百万像素。
- 倍数支持 16 / 32 / 64。
- 可选择是否同时处理 control 图。
- 处理完成后可直接载入为当前工作区。

10. Viewer 单条图像处理

- 支持当前条目缩放。
- 支持当前条目“匹配结果图尺寸”。

11. 图像预处理面板

- 已新增“批量匹配结果尺寸”。
- 会按每条 result 的尺寸把对应 control 批量对齐。
- 走统一任务进度状态。

12. 导出

- 支持 zip 或 folder。
- 支持用户自定义项目名。
- 默认非破坏式导出，不改原始目录。
- 图像可在导出时再次处理。
- 导出目录命名为“时间戳 + 项目名 + role”。

13. 数据目录统一

当前代码统一使用 datasets 体系：
- datasets/projects
- datasets/tmp
- datasets/exports
- datasets/processed
- datasets/workspaces

四、环境与启动方式

1. 启动方式

- 使用 ./start.sh 启动。
- start.sh 会：
  - 检查 Python
  - 自动创建当前目录下的 .venv
  - 安装基础依赖
  - 启动 web_server.py

2. 环境脚本

- tools/bootstrap_env.py 已实现跨平台 venv 逻辑。
- 已补充常见错误诊断，包括：
  - Python 缺失
  - Python 版本不匹配
  - venv 模块缺失
  - 网络失败
  - SSL 证书问题
  - 磁盘空间不足
  - 权限问题

3. 重启方式

- Linux 下可用：
  KILL_EXISTING=1 ./start.sh
  自动杀掉占用端口的旧进程并重启。
